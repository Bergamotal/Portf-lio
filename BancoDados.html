Um banco de dados é uma coleção organizada de dados, projetada para facilitar o armazenamento, a recuperação e a gestão eficiente de informações. Utilizado em diversas aplicações, desde pequenas empresas até grandes corporações, um banco de dados permite a estruturação e o acesso a dados de maneira eficiente e segura.




Dados:

Os dados são os caracteres(nada mais e nada menos que as letras, número, símbolos e etc).


Informações:

As informações são as junçôes de vários dados, então, a junção de várias letras e/ou números formam uma ou diversas informações.


Conhecimento:

Já o conhecimento, é a junção de diversas informções que traz certo conhecimento à tona.


Inteligência:

Por fim a inteligência, que nada mais é que a junção de vários conhecimentos(um bom exemplo de inteligência são as IA's, conhecidas como inteligência artificiais, que são diversos conhecimentos aglomerados em uma máquina).


-- Tipos de dados --




Dados Estruturados:

Os Dados Estruturados são organizados em um formato predefinido, como tabelas, onde cada entrada segue um padrão consistente. Eles são facilmente pesquisáveis e analisáveis usando técnicas tradicionais de banco de dados e ferramentas de análise de dados.


Dados Não-Estruturados:

Os Dados Não-Estruturados são informações que não seguem um formato predefinido ou organizado em um modelo rígido, tornando-os mais difíceis de pesquisar, organizar e analisar em comparação com dados estruturados. Esses dados são geralmente textos ou multimídia e não se encaixam facilmente em tabelas relacionais ou outros formatos de dados estruturados.


Dados Semi-Estruturados:

Os Dados Semi-Estruturados são um tipo de dados que não são organizados em um esquema rígido de tabelas, como os dados estruturados, mas contêm tags ou marcadores para separar elementos e impor uma hierarquia de registros e campos dentro dos dados. Isso permite uma flexibilidade maior na forma como os dados são armazenados e processados, enquanto ainda mantém alguma estrutura que facilita a análise.



-- Tipos de Armazenamento --



Antes de falar dos tipos de armazenamentos vamos ver o que é um servidor:



Um servidor é um computador ou sistema de software que fornece serviços, recursos, ou dados a outros computadores, conhecidos como clientes, através de uma rede. Os servidores podem executar uma variedade de funções, desde hospedar websites até gerenciar banco de dados e fornecer serviços de email.



Agora os armazenamentos, eles são separados em 3 tipos:


Privado:

O armazenamento privado é um ambiente de armazenamento dedicado exclusivamente a uma única organização, e não é compartilhado com outras entidades.


Públicos:

O armazenamento público é um serviço oferecido por provedores de nuvem, onde os recursos de armazenamento são compartilhados entre vários clientes. Os dados são armazenados em nuvens e acessados pela internet.


Híbridos:

O armazenamento híbrido combina recursos de armazenamento privado e público, permitindo que as organizações utilizem ambos conforme necessário. Oferece a flexibilidade de armazenar dados sensíveis em armazenamento privado e dados menos críticos em armazenamento público.

-- Modelo entidade relacionamento(ER) --


Em um e-commerce, há duas classes de dados mantidos em um banco de dados. Segue uma breve descrição de cada umas destas classes de dados:


Entidade Cliente:

É quem de fato efetua as compras dentro do sistemas. Seus campos são: código do cliente, primeiro nome, último nome, cpf e endereço.


Entidade Produto:

Armazena os dados de cada produto disponível no estoque do e-commerce. Seus campos são: código do produto, nome do produto, marca, categoria e preço.


Relacionamento:

Estas duas entidades do sistema se relacionam da seguinte forma: Um cliente pode comprar no mínimo 0 produtos ou no máximo N produtos. Já um produto pode ser comprado por no mínimo 0 cliente ou no máximo por 1 cliente.

-- Modelo conceitual - DER(Diagrama entidade relacionamnto) --



PRIMARY-Key:

valor único e não ausente que representa cada item salvo no Banco de Dados.

FOREIGN-Key:

é a primary key importada de outra entidade que estabelece relacionamento entre as entidades.

REGRA:

Em um relacionamento 1 para N, a foreign key fica sempre do lado N.


-- PostGreSQL --


O PostgreSQL, frequentemente abreviado como Postgres, é um sistema de gerenciamento de banco de dados relacional de código aberto altamente robusto e avançado. Conhecido por sua confiabilidade, integridade dos dados e conformidade com os padrões SQL, o PostgreSQL é amplamente utilizado em aplicações que requerem complexidade de dados e operações transacionais sofisticadas.





-- SQL --

SQL (Structured Query Language) é uma linguagem de programação usada para gerenciar e manipular dados em sistemas de gerenciamento de banco de dados relacionais (SGBDR).

-- Normalização --


Normalização é o processo de organização de dados em um banco de dados. Isso inclui a criação de tabelas e o estabelecimento de relações entre essas tabelas de acordo com as regras projetadas para proteger os dados e tornar o banco de dados mais flexível, eliminando a redundância e a dependência inconsistente. Dados redundantes desperdiçam espaço em disco e criam problemas de manutenção. Se os dados existentes em mais de um local devem ser alterados, eles devem ser alterados exatamente da mesma maneira em todos os locais. Uma alteração de endereço do cliente é muito mais fácil de implementar se esses dados são armazenados apenas na tabela Clientes e em nenhum outro lugar no banco de dados.
